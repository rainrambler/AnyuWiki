- 本节介绍用于应对移动安全威胁的安全机制。它们被分为不同的类别，因为它们的行为不同，它们的范围从操作系统的安全管理到用户的行为教育。各种措施所阻止的威胁因情况而异。考虑到上面提到的两种情况，在第一种情况下，可以保护系统免受应用程序的破坏，在第二种情况下，可以防止安装可疑软件。
- 操作系统中的安全性
	- 智能手机的第一层安全性是操作系统 (OS)。除了需要处理设备上操作系统的常见角色（例如资源管理、调度进程）之外，它还必须建立引入外部应用程序和数据的协议，而不会引入风险。
- 移动操作系统的核心范式是沙箱([[Sandbox]])的概念。由于智能手机目前设计用于容纳许多应用程序，因此它们必须具有确保这些应用程序对手机本身、系统上的其他应用程序和数据以及用户安全的机制。如果恶意程序到达移动设备，系统呈现的易受攻击区域必须尽可能小。沙盒将这一想法扩展到划分不同的进程，防止它们相互作用和相互破坏。根据操作系统的历史，沙盒有不同的实现。例如，默认情况下，iOS 将专注于限制对来自 App Store 的应用程序的公共 API 的访问，而 Managed Open In 允许您限制哪些应用程序可以访问哪些类型的数据。 [[Android]] 的沙盒基于其遗留的 Linux 和 TrustedBSD。
- 以下几点重点介绍了在操作系统中实现的机制，尤其是 Android。
	- [[Rootkit]]检测器
		- rootkit 侵入系统是一个巨大的危险，就像在计算机上一样。防止此类入侵并能够尽可能频繁地检测到它们非常重要。事实上，人们担心这种类型的恶意程序可能会导致部分或完全绕过设备安全，并让攻击者获得管理员权限。如果发生这种情况，那么没有什么可以阻止攻击者研究或禁用被规避的安全功能，部署他们想要的应用程序，或者将 rootkit 的入侵方法传播给更广泛的受众。作为防御机制，我们可以引用 iOS 中的信任链([[Chain of trust]])。这种机制依赖于启动操作系统所需的不同应用程序的签名，以及 Apple 签署的证书。如果签名检查不确定，设备会检测到这一点并停止启动。如果操作系统因越狱而受到威胁，如果通过越狱([[Jail-breaking]])方法禁用它或在越狱禁用 Rootkit 检测后加载软件，则 Rootkit 检测可能无法正常工作。
	- 进程隔离
		- Android 使用继承自 Linux 的用户进程隔离机制。每个应用程序都有一个与之关联的用户和一个元组（[[UID]]，[[GID]]）。这种方法充当沙箱：虽然应用程序可能是恶意的，但它们无法离开为其标识符保留的沙箱，因此不会干扰系统的正常运行。例如，由于一个进程不可能结束另一个用户的进程，因此应用程序不能停止另一个用户的执行。
	- 文件权限
		- 在 Linux 的遗留系统中，还有文件系统权限([[filesystem permissions]])机制。它们有助于沙盒：一个进程不能编辑它想要的任何文件。因此，不可能随意损坏其他应用程序或系统运行所需的文件。此外，在 Android 中还有锁定内存权限的方法。无法从手机更改安装在 SD 卡上的文件的权限，因此无法安装应用程序。
	- 内存保护
		- 与在计算机上一样，内存保护可以防止特权升级([[Privilege escalation]])。实际上，如果一个进程设法到达分配给其他进程的区域，它可以写入具有高于自己的权限的进程的内存，在最坏的情况下具有root权限，并执行超出其系统权限的操作。插入由恶意应用程序的权限授权的函数调用就足够了。
	- 通过运行时环境进行开发
		- 软件通常是用高级语言开发的，它可以控制正在运行的程序正在做什么。例如，Java 虚拟机持续监控它们管理的执行线程的动作，监控和分配资源，并防止恶意动作。这些控件可以防止缓冲区溢出。
		  安全软件
- 安全软件
	- 在操作系统安全之上，还有一层安全软件。该层由单个组件组成，以加强各种漏洞：防止恶意软件、入侵、将用户识别为人类以及用户身份验证。它包含从他们的计算机安全经验中学到的软件组件；但是，在智能手机上，该软件必须处理更大的限制。
	- 防病毒和防火墙
		- 可以在设备上部署防病毒软件，以验证它没有被已知威胁感染，通常是通过检测恶意可执行文件的签名检测软件。与此同时，防火墙可以监视网络上的现有流量，并确保恶意应用程序不会试图通过它进行通信。它同样可以验证已安装的应用程序不会寻求建立可疑通信，这可以防止入侵尝试。
		- 移动防病毒产品将扫描文件并将它们与已知的移动恶意软件代码签名数据库进行比较。
	- 视觉通知
		- 为了让用户意识到任何异常行为，例如他们没有发起的呼叫，可以将一些功能链接到无法规避的视觉通知。例如，当触发呼叫时，应始终显示被叫号码。因此，如果呼叫是由恶意应用程序触发的，用户可以看到并采取适当的措施。
	- 图灵测试
		- 与上述相同，重要的是通过用户决定确认某些操作。图灵测试用于区分人类和虚拟用户，它通常以验证码的形式出现。
	- 生物识别
		- 另一种使用方法是生物识别技术。生物识别技术是一种通过人的形态（例如，通过面部或眼睛的识别）或他们的行为（例如，他们的签名或书写方式）来识别人的技术。使用生物识别安全的一个优点是用户可以避免必须记住密码或其他秘密组合来验证和防止恶意用户访问他们的设备。在具有强大生物识别安全性的系统中，只有主要用户才能访问智能手机。
- 智能手机中的资源监控
	- 当应用程序通过各种安全障碍时，它可以执行其设计的操作。当触发此类操作时，如果监控手机上使用的各种资源，有时可以检测到恶意应用程序的活动。根据恶意软件的目标，感染的后果并不总是相同的；所有恶意应用程序都无意损害部署它们的设备。以下部分描述了检测可疑活动的不同方法。
	- 电池
		- 一些恶意软件旨在耗尽手机的能源资源。监控手机的能耗可以成为检测某些恶意软件应用程序的一种方式。
	- 内存使用情况
		- 内存使用在任何应用程序中都是固有的。但是，如果发现应用程序使用了相当大比例的内存，则可能会将其标记为可疑。
	- 网络流量
		- 在智能手机上，许多应用程序必须通过网络连接，作为其正常操作的一部分。但是，使用大量带宽的应用程序可能会被强烈怀疑试图传达大量信息，并将数据传播到许多其他设备。这种观察只允许怀疑，因为一些合法的应用程序在网络通信方面可能非常耗费资源，最好的例子是流媒体视频。
	- 服务
		- 可以监控智能手机的各种服务的活动。在某些时刻，某些服务不应处于活动状态，如果检测到某个服务，则应怀疑该应用程序。例如，用户在拍摄视频时发送短信：这种交流没有意义，而且很可疑；恶意软件可能会在其活动被屏蔽时尝试发送 SMS。
	- 上面提到的各个点只是指示，并不能确定应用程序活动的合法性。但是，这些标准可以帮助定位可疑应用程序，尤其是在结合多个标准的情况下。
- 网络监控
	- 可以监控电话交换的网络流量。可以在网络路由点中设置保护措施，以检测异常行为。由于移动设备对网络协议的使用比计算机更受限制，因此可以预测预期的网络数据流（例如用于发送 SMS 的协议），从而可以检测移动网络中的异常情况。
	- 垃圾邮件过滤器
		- 与电子邮件交换一样，我们可以通过移动通信（SMS、MMS）检测垃圾邮件活动。因此，可以通过部署在中继这些消息的网络基础设施上的过滤器来检测和最小化这种尝试。
	- 存储或传输信息的加密
		- 因为交换的数据总是有可能被截获，所以通信甚至信息存储都可以依靠加密来防止恶意实体使用通信期间获得的任何数据。然而，这带来了加密算法的密钥交换问题，这需要一个安全的通道。
	- 电信网络监控
		- SMS 和 MMS 网络表现出可预测的行为，与使用 TCP 或 UDP 等协议所能做的相比，没有那么多自由。这意味着人们无法预测网络通用协议的使用情况；一个人可能很少通过咨询简单的页面产生很少的流量，或者通过使用视频流产生大量流量。另一方面，通过手机交换的消息有一个框架和一个特定的模型，在正常情况下，用户没有自由干预这些通信的细节。因此，如果发现移动网络中的网络数据流出现异常，可以快速发现潜在威胁。
- 制造商监督
	- 在移动设备的生产和分销链中，制造商有责任确保设备以没有漏洞的基本配置交付。大多数用户不是专家，而且很多用户并不知道存在安全漏洞，因此制造商提供的设备配置将被许多用户保留。下面列出了制造商应该考虑的几点。一些智能手机制造商插入 Titan M2 以增强移动安全性。
	- 删除调试模式
		- 手机在制造过程中有时会设置为调试模式，但在销售手机之前必须禁用此模式。此模式允许访问不同的功能，而不是供用户日常使用。由于开发和生产的速度，会出现分心，一些设备以调试模式出售。这种部署使移动设备暴露于利用这种疏忽的漏洞。
	- 默认设置
		- 出售智能手机时，其默认设置必须正确，不得留下安全漏洞。默认配置并不总是更改，因此良好的初始设置对用户来说至关重要。例如，有些默认配置容易受到拒绝服务攻击。
	- 应用安全审计
		- 随着智能手机的出现，应用商店也应运而生。用户发现自己面临着大量的应用程序。对于管理应用商店的提供商来说尤其如此，因为他们的任务是从不同的角度（例如安全性、内容）检查所提供的应用程序。安全审计要特别小心，因为如果没有检测到故障，应用程序可以在几天内迅速传播，并感染大量设备。
	- 检测要求权限的可疑应用程序
		- 安装应用程序时，最好警告用户注意组合在一起的权限集，这些权限集似乎有潜在危险，或者至少是可疑的。 Android 上的 Kirin 等框架会尝试检测和禁止某些权限集。
	- 撤销程序
		- 除了应用商店之外，还出现了移动应用的一项新功能：远程撤销。该程序首先由 Android 开发，可以远程和全局卸载任何设备上的应用程序。这意味着当发现威胁时，可以立即阻止设法逃避安全检查的恶意应用程序的传播。
	- 避免高度定制的系统
		- 制造商很想在现有操作系统上覆盖自定义层，其双重目的是提供自定义选项和禁用某些功能或对其收费。这具有双重影响，即冒着在系统中引入新错误的风险，同时激励用户修改系统以规避制造商的限制。这些系统很少像原始系统那样稳定可靠，并且可能遭受网络钓鱼尝试或其他攻击。
	- 改进软件补丁流程
		- 定期发布智能手机各种软件组件的新版本，包括操作系统。随着时间的推移，他们纠正了许多缺陷。然而，制造商通常不会及时将这些更新部署到他们的设备上，有时甚至根本不会。因此，漏洞在可以被纠正时仍然存在，如果不纠正，因为它们是已知的，它们很容易被利用。
- 用户意识
	- 用户的粗心允许许多恶意行为。发现智能手机用户在应用程序安装过程中忽略了安全消息，尤其是在应用程序选择、检查应用程序信誉、评论以及安全和协议消息期间。从简单地在没有密码的情况下离开设备，到精确控制授予添加到智能手机的应用程序的权限，用户在安全循环中负有很大的责任：不要成为入侵的媒介。如果用户是在设备上存储业务数据的公司的员工，则此预防措施尤其重要。下面详细介绍了用户可以采取的一些预防措施来管理智能手机的安全性。
	- 互联网安全专家 BullGuard 最近的一项调查显示，对影响手机的恶意威胁数量不断增加缺乏洞察力，53% 的用户声称他们不知道智能手机的安全软件。另有 21% 的人认为这种保护是不必要的，42% 的人承认他们没有想到这一点（“使用 APA”，2011 年）。这些统计数据表明，消费者并不担心安全风险，因为他们认为这不是一个严重的问题。这里的关键是永远记住智能手机实际上是掌上电脑，同样容易受到攻击。
	- 持怀疑态度
		- 用户不应相信可能呈现的所有信息，因为某些信息可能是网络钓鱼或试图分发恶意应用程序。因此，建议在实际安装之前检查他们想要购买的应用程序的声誉。
	- 授予应用程序的权限
		- 应用的海量分发伴随着每个操作系统建立不同的权限机制。有必要向用户阐明这些权限机制，因为它们因系统而异，而且并不总是易于理解。此外，如果权限数量过多，则很少可能修改应用程序请求的一组权限。但最后一点是风险的来源，因为用户可以授予应用程序的权限，远远超出其所需的权限。例如，笔记应用程序不需要访问地理定位服务。用户在安装过程中必须确保应用程序所需的权限，如果请求的权限不一致，则不应接受安装。
	- 当心
		- 通过简单的手势和预防措施保护用户的手机，例如在不使用时锁定智能手机、不让其设备无人看管、不信任应用程序、不存储敏感数据或加密无法与设备分离的敏感数据。
	- 断开未使用的外围设备
		- NIST 2013 移动设备安全管理指南建议：限制用户和应用程序访问硬件，例如数码相机、GPS、蓝牙接口、USB 接口和可移动存储。
	- 确保数据
		- 智能手机具有很大的内存，可以承载数 GB 的数据。用户必须小心它携带的数据以及它们是否应该受到保护。虽然如果复制一首歌通常不会很戏剧化，但包含银行信息或商业数据的文件可能更具风险。用户必须谨慎避免在智能手机上传输敏感数据，这些数据很容易被盗。此外，当用户摆脱设备时，他们必须确保首先删除所有个人数据。
		- 这些预防措施是无法轻松解决智能手机中的人或恶意应用程序入侵的措施。如果用户小心，许多攻击都可以被击败，尤其是网络钓鱼和仅寻求获得设备权限的应用程序。
	- 短信集中存储
		- 一种形式的移动保护允许公司通过将消息托管在公司服务器上而不是发送者或接收者的电话上来控制文本消息的传递和存储。当满足某些条件时，例如到期日期，消息将被删除。

- 某些安全措施的限制
	- 本文中提到的安全机制在很大程度上继承了计算机安全方面的知识和经验。构成这两种设备类型的元素相似，并且有可以使用的通用措施，例如防病毒软件和防火墙。然而，这些解决方案的实现不一定是可能的，或者至少在移动设备内受到高度限制。造成这种差异的原因是计算机和移动设备提供的技术资源：尽管智能手机的计算能力越来越快，但除了计算能力之外，它们还有其他限制。
	- **单任务系统**：一些操作系统，包括一些仍然常用的，是单任务的。只执行前台任务。很难在此类系统上引入防病毒和防火墙等应用程序，因为它们无法在用户操作设备时执行监控，而此时最需要此类监控。
	- **能源自主性**：使用智能手机的一个关键是能源自主性。重要的是，安全机制不能消耗电池资源，否则设备的自主性将受到极大的影响，破坏了智能手机的有效使用。
	- **网络**：直接关系到电池寿命，网络利用率不宜过高。从能源消耗的角度来看，它确实是最昂贵的资源之一。尽管如此，为了节省电池，一些计算可能需要重新定位到远程服务器。这种平衡可以使某些密集计算机制的实现成为一个微妙的提议。
	- 此外，通常会发现存在更新，或者可以开发或部署更新，但并不总是这样做。例如，可以找到不知道有与智能手机兼容的较新版本操作系统的用户，或者用户可能会发现直到漫长的开发周期结束才被纠正的已知漏洞，这使得是时候利用漏洞了。
- **下一代移动安全**
- 预计将有四种移动环境构成安全框架：
- 丰富的操作系统
	- 这一类别将属于传统的移动操作系统，如 Android、iOS、Symbian OS 或 Windows Phone。它们将为应用程序提供操作系统的传统功能和安全性。
- 安全操作系统（安全操作系统）
	- 一个安全的内核，将在同一个处理器内核上与功能齐全的 Rich OS 并行运行。它将包括用于 Rich OS（“正常世界”）的驱动程序，以与安全内核（“安全世界”）进行通信。受信任的基础设施可以包括像显示器或键盘这样的接口到 PCI-E 地址空间和存储器的区域。
- 可信执行环境 ([[TEE]])
	- 由硬件和软件组成。它有助于控制访问权限并容纳需要与 Rich OS 隔离的敏感应用程序。它有效地充当“正常世界”和“安全世界”之间的防火墙。
- 安全元件 (SE)
	- SE 由防篡改硬件和相关软件或单独的隔离硬件组成。它可以提供高级别的安全性并与 TEE 协同工作。 SE 将是托管近距离支付应用程序或官方电子签名的强制性要求。 SE 可以连接、断开、阻止外围设备和操作单独的硬件集。
- 安全应用程序 (SA)
	- App Store 上有许多安全应用程序，提供病毒防护和漏洞评估服务。
-
